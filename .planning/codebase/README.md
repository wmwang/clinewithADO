# Codebase Mapping

This directory contains structured documents generated by the gsd-codebase-mapper agents. Each document provides insights into different aspects of the codebase, including technology stack, architecture, quality conventions, and concerns.

## Documents Overview

- **STACK.md**: Details the technology stack used in the project.
- **INTEGRATIONS.md**: Describes integrations with external services.
- **ARCHITECTURE.md**: Outlines the architectural design of the application.
- **STRUCTURE.md**: Provides an overview of the project structure.
- **CONVENTIONS.md**: Lists coding conventions and best practices.
- **TESTING.md**: Details the testing strategies employed.
- **CONCERNS.md**: Highlights any known concerns or issues within the codebase.

This mapping serves as a reference for understanding the current state of the codebase and guiding future development efforts.
